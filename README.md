# AI Engineering Brief

Static Astro site for the Wednesday and Sunday AI Engineering Brief editions generated by Hermes AI.

## What It Does

- Renders brief articles from `src/content/briefs/*.md` or `*.mdx`.
- Shows a mobile-friendly article list with 10 posts per page.
- Supports automatic dark/light mode with a manual toggle.
- Builds a fully static `dist` folder for Cloudflare Pages.
- Provides a publishing script that Hermes can run after each cron job.

## Local Setup

```bash
npm install
npm run dev
```

Build and check:

```bash
npm run build
```

Run Lighthouse CI after a build:

```bash
npm run lighthouse
```

If the machine does not have Chrome on `PATH`, set `CHROME_PATH` to a Chromium or Chrome executable.

## Publishing From Hermes

Hermes should gather the brief content first, then pass JSON into the publishing script.

Required JSON shape:

```json
{
  "title": "AI Engineering Brief: 2026-06-14",
  "summary": "Short reader-facing summary.",
  "date": "2026-06-14",
  "cadence": "Sunday",
  "body": "Markdown or MDX article body.",
  "sources": [
    { "title": "Source name", "url": "https://example.com" }
  ]
}
```

Publish from a file:

```bash
npm run publish:brief -- --input ./brief.json
```

Publish from a gather command:

```bash
BRIEF_GATHER_COMMAND="hermes brief --cadence Sunday --json" npm run publish:brief
```

The script writes an MDX file, builds Astro, deploys `dist` with Wrangler, then sends Telegram a HTML-formatted summary with the article link.

Useful flags:

```bash
npm run publish:brief -- --input ./brief.json --skip-deploy
npm run publish:brief -- --input ./brief.json --skip-telegram
npm run publish:brief -- --input ./brief.json --git
```

## Cloudflare Pages

Deploy with Wrangler:

```bash
npm run deploy:project
npm run build
npm run deploy
```

Wrangler 4.100 deploys Pages projects but does not expose a custom-domain subcommand. Add the custom domain from the Cloudflare dashboard, or call the Pages domains API with a token that can manage Pages and DNS:

```bash
curl --request POST \
  "https://api.cloudflare.com/client/v4/accounts/<ACCOUNT_ID>/pages/projects/ai-engineering-brief-site/domains" \
  --header "Authorization: Bearer <CLOUDFLARE_API_TOKEN>" \
  --header "Content-Type: application/json" \
  --data '{"name":"ai-news.leihuang.me"}'
```

The custom domain for this project is `ai-news.leihuang.me`; Cloudflare may keep it in `pending` while DNS and certificate validation complete.
